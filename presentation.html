背景与痛点

TEZ 运维的"信息孤岛"

腾讯云边缘可用区（Tencent Edge Zone）承载大量客户业务，但运维工具长期依赖人工 + 浏览器 + 电子表格

🔍

数据碎片化

查一台母机状态需要在 4 个系统（CMDB / TCUM / 数全通 / 运营总表）之间来回切换，每次 3-5 分钟

📋

工单靠聊天

搬迁、投放、故障等工单通过企业微信聊天流转，无记录、无追踪、无统计，事情靠人催

👤

找对接口人难

CMDB / TCUM / 数全通 / 搬迁 / 开区... 每个模块有不同对口人，新人一周才能搞清"这事该找谁"

📊

全局水位不可见

全网有多少空闲机位？哪些 Zone 即将饱和？全靠 Excel 手工统计，永远慢半拍

解决方案

聚合而非替代

不是再造一个 CMDB / 工单系统，而是把已有系统的能力聚合到一个统一入口，做轻、做快、做完

技术亮点

三项关键创新

🤖

浏览器自动化适配

当上游平台（CMDB/TCUM/IDCRM）没有 OpenAPI 时，通过 Playwright 自动化直接抓取数据，复用 SSO 登录态（7 天有效）

# 三态客户端模式 cmdb_mode: mock | api | browser tcum_mode: mock | api | browser idcrm_mode: mock | api | browser
⚡

智能分层缓存

三级缓存策略：Redis 热数据 → 本地 SQLite → 上游实时查询。缓存命中 <10ms，首次同步 15-30s

主机缓存 24h Zone 快照 7d 登录态 7d 并发限流 x5
🎯

AI 智能助手

集成大语言模型（混元/DeepSeek/GPT），支持三种上下文模式：竞争分析 · 运维知识 · 自由对话。从驾驶舱一键提问

竞争分析 运维知识 自由对话 流式响应
